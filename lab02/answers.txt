Section 14: 
Try to add a method addDigitalVideoDisc which allows to pass an arbitrary number of arguments for dvd. Compared to an array parameter. What do you prefer in this case? 

Ans: 
I prefer using the varargs approach (DigitalVideoDisc... dvds) because it provides greater flexibility and a cleaner syntax for the caller. It allows passing multiple objects directly without explicitly creating an array, while still preserving the method’s logic.

Section 15.1:
After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two 
objects still remain? 

Ans: 
The titles of the two objects remain unchanged because the swap method only exchanges local copies of the references. These changes do not affect the original references outside the method.

Section 15.2:
After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is 
the title of the JungleDVD changed? 

Ans: 
The title of jungleDVD is changed because the changeTitle method modifies the actual object through a setter. Since the reference points to the same object in memory, any change to its state is reflected outside the method.

-> Java is always a pass-by-value programming language.
